# Skill: Brainstorming

Use this skill when the user asks for ideas, UX direction, feature concepts, layout options, ecommerce flows, product presentation, promotions, or ways to make the clone feel more complete.

## Brainstorming rules

- Start broad, then narrow to practical implementation choices.
- Prefer ecommerce ideas that improve buying confidence, navigation, trust, or conversion.
- Separate must-have, should-have, and nice-to-have ideas.
- Avoid suggesting work that requires unauthorized scraping or proprietary assets.
- End with a recommended next step.

## Useful brainstorming categories

- Homepage sections.
- Product discovery and filters.
- Product detail conversion elements.
- Cart and checkout trust builders.
- Promotion mechanics.
- Mobile navigation.
- Empty/loading/error states.
- Visual design systems.
- Mock data strategy.

## Output format

For quick ideation, use:

1. Best recommendation.
2. Why it matters.
3. Implementation notes.
4. Tradeoff.

For deeper ideation, group ideas by priority and mark the first buildable slice.
