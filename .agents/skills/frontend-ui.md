# Skill: Frontend UI Builder

Use this skill when building or improving UI components, pages, layouts, product cards, navigation, cart, checkout, or responsive behavior.

## Inputs to gather

- Target page or component.
- Desired reference behavior or screenshot, if available.
- Data requirements and mock data shape.
- Breakpoints or devices that matter most.
- Whether implementation should be pixel-close or directionally similar.

## Process

1. Break the UI into sections and states.
2. Define the minimum data shape needed.
3. Build semantic structure first.
4. Apply responsive styling.
5. Add interactions only where needed.
6. Verify in browser.

## Output expectations

- Production-like responsive UI.
- Accessible controls and labels.
- Clean component boundaries.
- No unnecessary libraries.
- No proprietary asset copying.

## Inspiration from Stitch-style workflows

When the user asks for frontend generation or visual iteration, act like a UI prototyping partner:

- Generate a clear component/page direction quickly.
- Offer 2-3 layout alternatives when the design is ambiguous.
- Convert chosen direction into code with real responsive behavior.
- Preserve implementation quality instead of only producing a static mockup.
